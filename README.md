## 1. Core Architecture Specifications

### 1.1 CPU Bit Width

The system architecture is defined as **16-bit**. This standard was selected to balance computational capability with hardware feasibility for a single-developer implementation.

* **Data Path:** 16-bit (Process 2 bytes per cycle).
* **Performance Theory:** Compared to an 8-bit architecture running at an identical clock frequency, this design offers approximately double the data throughput per cycle.
* **Design Constraint:** All General Purpose Registers (GPRs) and internal buses are locked to a 16-bit width.

### 1.2 Endianness

The system utilizes **Big Endian** byte ordering.

* **Definition:** The Most Significant Byte (MSB) is stored at the lowest memory address.
* **Justification:** Big Endian is more intuitive for human readability during memory dumps and debugging (reading hex values from left to right matches the stored order).

## 3. Register File Organization

The CPU implements a streamlined register file to maximize instruction encoding efficiency.

### 3.1 General Purpose Registers (GPR)

The system contains **8 Registers** (R0 – R7).

* **Encoding:** 3 bits are required to address each register ().
* **Layout:**
* **R0–R6:** Standard General Purpose Registers for arithmetic and pointers.
* **R7 (Extension Register):** Reserved for **Widening Operations**. When performing arithmetic that exceeds 16 bits (e.g., multiplication results or overflow), R7 captures the "spillover" or high-order bits, effectively acting as the upper half of a 32-bit virtual register.

## 4. Instruction Set Architecture (ISA)

### 4.1 Instruction Encoding Format

The system utilizes a **Fixed-Length 16-bit Instruction Format**.

* **Efficiency:** A fixed width simplifies the hardware Control Unit (Fetch/Decode logic) significantly compared to variable-length CISC designs, reducing propagation delay and transistor count.
* **Structure:** Every instruction is exactly 1 Word (16 bits) long.

### 4.2 Bit Distribution (Three-Operand Layout)

To maintain a powerful instruction set within 16 bits, the following encoding scheme is adopted:

| Field | Bit Width | Description |
| --- | --- | --- |
| **Opcode** | **7 bits** | Defines the operation (Support for  unique instructions). |
| **Destination (Rd)** | **3 bits** | Target register for the result. |
| **Source 1 (Rs1)** | **3 bits** | First operand register. |
| **Source 2 (Rs2)** | **3 bits** | Second operand register. |

**Total Length:** **16 bits**

### 4.3 Example Assembly

```asm
ADD R1, R2, R3   ; R1 = R2 + R3

```

* **Opcode (ADD):** 7 bits
* **Rd (R1):** 001
* **Rs1 (R2):** 010
* **Rs2 (R3):** 011

## 6. Bus Protocol (The Handshake)

Since the CPU is 16-bit and the address is 24-bit, the "Conversation" between the CPU and the RAM needs a strict protocol.

### 6.1 The Signal Lines

To talk to the memory, the CPU needs these physical pins:

1. A0–A23 (24 pins): Address Bus.

2. D0–D15 (16 pins): Data Bus.

3. RW (1 pin): Read/Write signal (High = Read, Low = Write).

4.  CE (1 pin): Chip Enable (Tells the RAM "I am talking to YOU").

5. CLK (1 pin): System Clock to keep everyone in sync.

There are three main reasons for this:

    Electrical Stability (The "Pull-up" Reason): If a wire accidentally breaks or a chip is disconnected, a "Pull-up resistor" ensures the wire stays at a High voltage (1). If the signal were Active-High, a broken wire might look like a "Disable" signal. Since "High" is the default resting state, we use "Low" (connecting to ground) to "Trigger" the action.

    Current Sinking: Historically, it was easier for transistors to "sink" current (pull a signal to 0V) than to "source" it (push it to 5V).

    Wired-OR Logic: You can connect multiple "Active-Low" outputs together more easily. If any one chip pulls the line Low, the signal is active.

### 6.2 The "Read" Cycle (How the CPU gets data)

1. T1 (Clock Cycle 1): CPU puts the 24-bit address on the Address Bus.

2. T2 (Clock Cycle 2): CPU sets RW to High (Read) and pulls CE Low (Enable).

3. T3 (Clock Cycle 3): The RAM finds the data and places it on the Data Bus.

4. T4 (Clock Cycle 4): The CPU "latches" (grabs) the data from the bus and brings it inside a register.
